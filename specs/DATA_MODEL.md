Simple schema — no auth, no users, no RLS. Results are public by design.

## Tables

### analyses
The primary table. One row per repo analysis.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | text | PK | Short URL-safe ID (nanoid, 10 chars) |
| repo_url | text | NOT NULL | Full GitHub repo URL |
| repo_owner | text | NOT NULL | GitHub owner/org |
| repo_name | text | NOT NULL | Repository name |
| repo_branch | text | | Branch analyzed (null = default) |
| status | text | NOT NULL, DEFAULT 'pending' | pending, analyzing, complete, error |
| slop_score | integer | | Composite score 0-100 |
| scores | jsonb | | Per-dimension scores (see below) |
| verdict | text | | AI-generated verdict copy |
| receipts | jsonb | | Structured findings with fix prompts |
| metadata | jsonb | | Repo stats: file count, LOC, languages, etc. |
| error_message | text | | Error details if status = 'error' |
| analyzed_at | timestamptz | | When analysis completed |
| created_at | timestamptz | NOT NULL, DEFAULT now() | When request was created |

```sql
create table public.analyses (
  id text primary key,
  repo_url text not null,
  repo_owner text not null,
  repo_name text not null,
  repo_branch text,
  status text not null default 'pending' check (status in ('pending', 'analyzing', 'complete', 'error')),
  slop_score integer check (slop_score >= 0 and slop_score <= 100),
  scores jsonb,
  verdict text,
  receipts jsonb,
  metadata jsonb default '{}'::jsonb,
  error_message text,
  analyzed_at timestamptz,
  created_at timestamptz default now() not null
);

-- Index for result page lookups
create index analyses_id_idx on public.analyses(id);

-- Index for checking existing analyses of same repo
create index analyses_repo_idx on public.analyses(repo_owner, repo_name);

-- Index for polling by status
create index analyses_status_idx on public.analyses(status) where status in ('pending', 'analyzing');
```
